# Reproducible builds — locked requirement sets

NTH DAO's core package depends on the Python standard library only.
The optional `[crypto]`, `[lan]`, `[ux]`, `[web]`, and `[dev]` extras
pull in third-party packages. Reproducibility means a fresh checkout
plus a single `pip install` command MUST yield the same dependency
versions every time, on every machine.

## Layout

```
requirements/
├── README.md             — this file
├── base.txt              — empty (core is stdlib-only); kept for symmetry
├── crypto.lock.txt       — PyNaCl pinned + transitive deps
├── lan.lock.txt          — zeroconf pinned (when implementing v0.9.5+ mDNS)
├── ux.lock.txt           — qrcode + pillow pinned
├── web.lock.txt          — fastapi + uvicorn pinned + transitive deps
├── dev.lock.txt          — pytest + ruff + mypy pinned + transitive deps
└── all.lock.txt          — union of every extra (the demo / CI environment)
```

Each `*.lock.txt` is generated by:

```bash
pip-compile requirements/<name>.in -o requirements/<name>.lock.txt
```

If you don't have `pip-tools` installed, regenerate via:

```bash
pip install pip-tools
pip-compile requirements/all.in -o requirements/all.lock.txt
```

## Use from a fresh checkout

```bash
# Production install (no dev tools)
pip install -r requirements/all.lock.txt
pip install -e . --no-deps

# Dev install
pip install -r requirements/dev.lock.txt
pip install -e ".[crypto,web,ux,dev]"
```

## Why we do this

- Reviews caught six critical bugs in v0.9.0. Some were related to
  library behavior (e.g., Windows `ConnectionResetError` semantics in
  `socket` differ between Python versions). Pinning prevents
  upstream-version surprises in CI.
- A future contributor on a different machine MUST be able to reproduce
  the test suite bit-for-bit.
- Security review needs to know "what code was actually shipped" —
  unpinned `>=X` ranges make that question unanswerable.

## Rule

CI rebuilds these locks once per quarter (or whenever someone bumps an
upper bound deliberately). Between rebuilds the locks are immutable.
Any PR that edits a lock without rationale is rejected.
